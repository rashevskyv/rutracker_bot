# Результати виконання — Безпечне злиття Gist-стану та виправлення дайджесту (v0.6.37)

Усі заплановані зміни щодо безпечного злиття Gist-стану та виправлення помилки Telegram API `Unsupported start tag` було успішно реалізовано локально, а версії файлів об'єднано та завантажено.

## Зміни, що були внесені (v0.6.37)

### 1. Безпечне злиття при синхронізації Gist (`sync_gist_state.py`)
- Замінено логіку прямого перезапису файлів у Gist (`upload`) на інтелектуальне злиття (merge).
- Тепер під час завантаження стану локального сховища у Gist скрипт спочатку завантажує поточну версію файлів із хмари та об'єднує їх із локальними версіями:
  - **`manual_releases.json`**: Зливає списки ручних релізів. Якщо реліз присутній в обох файлах, зберігається позначка `"processed": true`, якщо хоча б одна з копій була оброблена на сервері. Поля релізів об'єднуються без втрати інформації.
  - **`posted_links.json`**: Об'єднує словники надісланих лінків, залишаючи останні часові мітки для кожної ссилки.
  - **`daily_digest_data.json` / `homebrew_digest_data.json`**: Об'єднує черги дайджестів, залишаючи найновіші записи за унікальними ключами (URL + статус оновлення).
  - **`last_digest_run.json` / `last_homebrew_digest_run.json`**: Залишає найновішу мітку часу останнього успішного запуску.
  - **`hb_state.json`**: Об'єднує версії хоумбрю-додатків, залишаючи записи з найбільшою версією чи найновішою датою оновлення.
  - **Для не-JSON файлів** (наприклад, `last_entry.txt`): Надає пріоритет версії з Gist (яка є актуальним станом з боку сервера).
- Злиті файли автоматично перезаписуються локально, а потім надсилаються в Gist. Це запобігає випадковому стиранню нових ручних релізів або міток часу відправки.

## Попередні зміни (v0.6.36)

### 1. Виправлення логіки обрізання HTML-описів (`main.py`)
- Змінено логіку очищення незавершених тегів при обрізанні `update_description` (опису оновлень) до 200 символів.
- Додано універсальний паттерн `r'<[^>]*$'`, який повністю вилучає будь-який недописаний тег наприкінці рядка.
- Додано автоматичне закриття тегу `</a>` шляхом додавання закривачого тегу, якщо після обрізання відкриваючий тег залишився без пари (зберігає посилання).

### 2. Очищення локального реєстру (`data/daily_digest_data.json`)
- Виправлено зламаний тег у грі `Disney Dreamlight Valley` (опис закінчувався на `<` замість `</a>`).

## Попередні зміни (v0.6.35)
