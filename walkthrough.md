# Результати виконання — Строгий фільтр ігор та свіжості релізів у кастомному колекторі (v0.6.50)

Реалізовано строгий фільтр для кастомного колектора `collect_custom_releases.py`: тепер збираються **виключно грабельні ігри/порти ігор** і **виключно свіжі релізи** (опубліковані за останні 3 дні).

## Зміни, що були внесені (v0.6.50)

### 1. Фільтрація не-ігрових компонентів та системних файлів
- Створено список виключень `EXCLUDE_KEYWORDS` (`kernel`, `l4t`, `nvgpu`, `dtb`, `firmware`, `driver`, `hekate`, `atmosphere`, `payload`, `cheat`, `patch`, `sysmodule`, `overlay` тощо).
- Додано метод `is_system_or_kernel_component`, який миттєво відсіює системні утиліти, ядра та прошивки без зайвих звернень до AI.
- Оновлено промпт для **Gemini 3.5 Flash thinking**: AI суворо класифікує репозиторій як `is_switch_game` лише у випадку, якщо це повноцінна гра або порт гри на Nintendo Switch.

### 2. Фільтрація за датою релізу (свіжість)
- Додано метод `is_release_recent` з обмеженням `MAX_RELEASE_AGE_DAYS = 3`.
- Скрипт перевіряє дату публікації тегу/релізу (`published_at` або `pushed_at`). Старі репозиторії та релізи з минулих місяців/років пропускаються.

### 3. Очищення стану та синхронізація Gist
- З файлу `data/manual_releases.json` вилучено попередні тестові не-ігрові записи (`DrasticDS`, `shield-l4t-kernel-nvgpu`, `shield-l4t-kernel-nvidia`).
- Виконано синхронізацію з GitHub Gist (`python sync_gist_state.py upload`).

### 4. Версіонування та документація
- Версію програми оновлено до `v0.6.50` у файлах `CHANGELOG.md`, `task.md` та `walkthrough.md`.
