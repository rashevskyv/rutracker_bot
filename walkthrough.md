# Результати виконання — Перевірка оновлень оброблених ручних релізів (v0.6.47)

Усі заплановані дії з розширення логіки `collect_homebrew_updates.py` для відстеження оновлень раніше оброблених ручних релізів було успішно виконано локально.

## Зміни, що були внесені (v0.6.47)

### 1. Перевірка оновлень ручних релізів у Homebrew Collector
- Модифікувати метод `load_homebrew_list` у `collect_homebrew_updates.py`:
  * Тепер колектор автоматично зчитує всі оброблені записи з `data/manual_releases.json` (де `processed` є `true`).
  * Для кожного такого запису витягується `api_url` GitHub-репозиторію та додається до загального списку відстеження.
  * Прапорець `new` для цих елементів встановлюється в `False`.
- Логіка виявлення та публікації оновлень:
  * Коли автор релізу публікує новий тег/реліз на GitHub (наприклад, `release3` замість `release2`), колектор фіксує це як оновлення.
  * Публікація в дайджесті відбувається з маркером `•` (як оновлення існуючої програми), а не `⚠️` (як для абсолютно нових застосунків).
  * При виявленні оновлення автоматично оновлюються поля `version`, `release_url` та `date` як у `data/hb_state.json`, так і у `data/manual_releases.json`.

### 2. Версіонування та документація
- Версію програми оновлено до `v0.6.47` у файлах `CHANGELOG.md`, `task.md` та `plan.md`.
- Суть змін детально задокументована у цьому файлі `walkthrough.md`.
